## **Section 2: Data Usage & Iteration Mapping**

This section describes how each data source is used within the application, what part of the system it supports, and which iteration it is introduced in. This ensures traceability between data decisions and product requirements.

| Source | Iteration | How Data is Used | Part of System | Notes |
| :---- | :---- | :---- | :---- | :---- |
| ARPANSA UV Index | Iteration 1 | Called directly via API on page load to retrieve the current real-time UV Index for Melbourne. The returned UV Index value is the core input that drives all safety alerts and recommendations displayed to the user. If UV Index \>= 3, the app triggers sun protection alerts with contextual advice. | Frontend display layerBackend API callAlert logic  | Direct API call — no local data storage in Iteration 1\.Must notify ARPANSA of app URL before go-live.Attribution must appear in the UI. |
| Sanchez-Perez et al. (2019) | Iteration 1 | UVI-to-sunburn probability thresholds from this paper are hardcoded into the alert logic. Used to determine at which UV Index value each skin type faces risk of first-degree sunburn, informing the personalised alert messages shown to users who indicate their skin type. | Alert logicPersonalised recommendations | Static reference — values hardcoded at build time.No runtime data fetch required.Cite the paper in the app About/data sources section. |
| ABS Sun ProtectionBehaviours Survey | Iteration 1 | Used during the Analysis & Design phase to understand the target audience's existing sun protection behaviours, sunburn rates by age group, and demographic risk profiles across Australia. Informs persona development, user story prioritisation, and the tone and content of safety messaging in the app. | A\&D phase:\- Persona profiles\- User story mapping\- Content & tone design\- Ethics canvas | Not ingested into the live app.Used as an evidence base for design decisions only.Cite in the persona documentation and ethics canvas. |
| Cancer CouncilSun Protection Guidelines | Iteration 1 | Provides the authoritative dosage thresholds and SunSmart recommendations that are used to write the app's alert copy and recommendation text. All user-facing advice (UV \>= 3 threshold, SPF50+ guidance, reapplication intervals, 5mL dosage) must align with these national guidelines. Also used to validate that alert logic is clinically appropriate. | A\&D phase:\- Content & alert copy design\- Ethics canvasBuild phase:\- Alert threshold validation\- Recommendation text | Reference only — not ingested.All advice shown to users must match current Cancer Council guidelines.Recheck guidelines before each iteration build in case of updates. |

*Note: This is a live document and will be updated at the start of each iteration to reflect any new data sources, changes to usage, or updated licensing terms. All feedback from studio mentors must be implemented before the next iteration build submission.*
